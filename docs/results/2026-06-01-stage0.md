# Stage 0 Results — Macro Deep-Search Distillation

**Date:** 2026-06-01 · **Status:** ✅ pipeline proven end-to-end · **Next:** Stage 1 (scale-up)

## Goal of Stage 0
Prove the *whole* pipeline works end-to-end on real infra — generate → filter → dataset →
train → serve → eval — and that train/serve parity holds. **Not** to produce a good model
(that's Stage 1); the point was to de-risk every moving part cheaply.

## What ran
On a RunPod **H100 80GB** pod:
1. **Generate** — 2 teacher traces with **gpt-5** through the shared `run_agent` harness
   (Perplexity Sonar `web_search` + `fetch_url` + FRED), visible-`Thought:` ReAct, forced
   finalization on the last step. Both produced ~4.5k-char structured reports.
2. **Filter** — mechanical gates + **gpt-5-mini** judge → kept 2/2.
3. **Dataset** — converted to records; loss mask verified by `render_and_mask` against the real
   Qwen3 template (only reasoning + tool calls + final report get loss).
4. **Train** — QLoRA on **Qwen3-8B** (4-bit nf4, LoRA r=32 → 87M trainable params = 1.05%).
   Loss **1.63 → 1.57 → 1.29 → 1.15** over 2 epochs. Adapter saved.
5. **Serve + eval** — student run through the **same** `run_agent` via a no-vLLM `HFDriver`
   (transformers + adapter, Qwen3 `<tool_call>` parser). It planned, called Sonar+FRED,
   recovered from a tool error, and wrote a report.

### Reading it
- **Frontier (gpt-5) leads on quality:** 4.25 vs student 3.13 vs base 2.88 (~1.1-pt gap to close).
  It wins by being thorough (uses the full 6-step budget; student/base stop ~3).
- **Even at 2 traces, the student already shows the distilled behaviors:** perfect tool-call
  validity (1.0 vs base 0.5) and **zero hallucinated numbers** (vs base 0.5) — matching the teacher
  on mechanics + grounding. The remaining gap is research *depth/quality*.
- **Caveat — anecdote, not proof:** n=2 traces, n=2 questions, high variance (student beat base on
  Q0, lost on Q1). Qwen3-8B base is already capable, so the bar isn't low.

## Key decisions made during execution (deviations from the plan)
- **transformers pinned `<5`** — 5.x breaks AutoTokenizer/GenerationMixin under torch 2.4 on GPU
  (worked on the Mac only because no torch → tokenizer-only mode). Got 4.57.6.
- **Dropped LLaMA-Factory for training → direct PEFT/transformers QLoRA** (`train/train_qlora.py`).
  The Stage-0 template-parity gate showed LLaMA-Factory's `sharegpt` tool format can't hold our
  "Thought + tool_call in one turn" (its `function_call` turns drop the reasoning — the distilled
  signal). We tokenize with the verified `render_and_mask` instead → parity is structural.
- **web_search backend = Perplexity Sonar** (replaced Tavily) — maps `search_results` to the same
  `{title,url,snippet}` shape; `format_search_results` unchanged.
- **Forced finalization** — gpt-5 never self-terminates within budget; reserve the last step to
  inject a "write the Final report now" instruction with tools disabled. Same at train + inference.
- **`HFDriver`** — a no-vLLM serving/eval path (base+adapter via transformers, `<tool_call>` parser).

## Stage-1 refinements (surfaced by Stage 0)
1. **Reasoning format parity:** student emits Qwen3 native `<think>…</think>` at inference while
   training data uses plain `Thought:` in content. Both work, but standardize (author teacher traces
   in `<think>` form, or disable thinking at inference) so trained ≈ served reasoning exactly.
2. **FRED series ids:** student guessed `FEDFUND` (→400) vs correct `DFF`/`FEDFUNDS`. More traces +
   a fred_search-first nudge, and/or make `fred_series` fall back to `fred_search` on a 400.
3. **Bigger eval set** (30–50 Qs) for statistically meaningful numbers; consider pairwise judging.
4. **Cost control:** run generation on the Mac (API-bound, no GPU) with the pod stopped; resume the
   pod only for train + eval (H100 training is minutes).
